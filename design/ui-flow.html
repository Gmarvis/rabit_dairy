03

How it's built underneath

You asked for an Expo monorepo with a DDD / CQRS approach on Supabase. Here's how those pieces map onto the screens above — kept light, so the UI stays the star.

📱 App · Expo

One React Native app, iOS + Android from a single codebase. expo-router for the tab + stack navigation you see in the mockups; expo-av for voice recording; expo-camera + expo-image-picker for scan & upload.

📦 Monorepo

An npm-workspaces monorepo:

  • apps/mobile — the Expo app
  • packages/domain — entities, value objects, rules
  • packages/application — commands & queries (CQRS)
  • packages/infra — Supabase adapters

🧩 DDD & CQRS

Commands (LogTransaction, ImportStatement, SetBudget) write; Queries (GetMonthlyReport, GetBudgetVsActual) read. The Transaction, Account, Budget, and Category aggregates hold the rules your spreadsheet encoded in formulas. Each transaction references an Account, so posting a deposit updates that account's balance and the dormant one stays out of totals.

⚡ Supabase

Postgres with Row-Level Security per user; Auth for email/phone/OAuth; Storage for voice notes and statement images; Edge Functions for OCR parsing and transcription. Offline-first, syncing when back online.

What the mockups deliberately don't show yet: exact OCR accuracy per bank/MoMo format, and the on-device vs. server split for transcription. Both depend on which providers we wire in — I've drawn the review step (screen 10) so that you always confirm before import, which keeps the app trustworthy regardless of how good the parsing gets.
